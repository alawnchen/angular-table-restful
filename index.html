Angular-table-restful

A table for AngularJs with support for restful API standard.

Download this project as a .zip file Download this project as a tar.gz file

at-table

The attribute that will receive the list from the controller. e.g.:

<table at-table="vm.list">
    <tbody>
        <tr>
            ...
        </tr>
    </tbody>
</table>

at-attribute

This attribute is responsible for binding the attribute to a header, creating the header automatically. e.g.:

<table at-table="vm.list" at-paginated>
  <tbody>
    <tr>      
      <td at-attribute="name"></td>
    </tr>
  </tbody>
</table>

at-sortable

This attribute is responsible for enabling sort to the specific column. e.g.:

<table at-table="vm.list" at-paginated>
  <tbody>
    <tr>      
      <td at-attribute="name" at-sortable></td>
    </tr>
  </tbody>
</table>

at-title

This attribute is responsible to add the header name for a specific column. In case its not defined the at-attribute name will be used as the header name. e.g.: 
<table at-table="vm.list" at-paginated>
  <tbody>
    <tr>      
      <td at-attribute="name" at-sortable at-title="Custom Name"></td>
    </tr>
  </tbody>
</table>

at-paginated

The attribute responsible for the pagination and the amount of itens to display per page (default 10). You can change the default value like this: at-paginated="5"

If you remove it, the pagination won't be rendered. e.g.:

<table at-table="vm.list" at-paginated> <!-- or at-paginated="5" to display 5 items per page -->
    <tbody>
        <tr>
            ...
        </tr>
    </tbody>
</table>

at-change

The attribute responsible to trigger the API Pagination. It receives a function to be called when the page/sort changes. e.g.:

HTML

<table at-table="vm.list" at-paginated at-change="vm.changePage">
    <tbody>
        <tr>
            ...
        </tr>
    </tbody>
</table>

Javascript

angular.module("angular-table-restful-example").controller("apiCtrl", ["$http", function($http) {

  vm.changePage = changePage;

                                        //make API call here...
  }
}])

at-load

If this attribute is used it will trigger the at-change function when the table is loaded for the first time loading the first page, so you won't need to load it manually. e.g.:

<table at-table="vm.list" at-paginated at-load at-change="vm.changePage">
    <tbody>
        <tr>
            ...
        </tr>
    </tbody>
</table>

at-pages-to-show

Number of pages that will be displayed in the pagination (default: 5). e.g.:

<table at-table="vm.list" at-paginated at-pages-to-show="10">
    <tbody>
        <tr>
            ...
        </tr>
    </tbody>
</table>

at-scroll

This attribute is responsible for disabling the horizontal scroll that appears when the page is too small(phones/tablets). By default the table has this enabled for a responsive layout. e.g.:

<table at-table="vm.list" at-paginated at-scroll="false">
    <tbody>
        <tr>
            ...
        </tr>
    </tbody>
</table>

at-ellipsis

When the content of a cell is too large the ellipsis gets triggered by default, adding a '...' at the end of that cell. The entire text is displayed when the user pass the mouse over the cell.

For this to work you must add some css to your project. e.g.:

HTML

<table at-table="vm.list" at-paginated at-ellipsis="false">
    <tbody>
        <tr>
            ...
        </tr>
    </tbody>
</table>

CSS

.text-ellipsis {
                                        max-width: 250px;
                                        white-space: nowrap;
                                        overflow: hidden;
                                        text-overflow: ellipsis;
        }
                                        .text-ellipsis:hover {
                                        text-overflow: initial;
                                        white-space: normal;
            }

at-ignore-header

This attribute is responsible for unbinding the at-attribute header. When this attribute is specified it will render only the thead that is written explicitly in the html. e.g.: 
<!-- Keep in mind that you can add ng-if/ng-show or anything that you want to make the table that matches your needs -->
<table at-table="vm.list">
    <thead at-ignore-header>
        <tr>
            <th> <!-- Some custom header, like a checkbox to check all items or make a group header separator--></th>
        </tr>        
    </thead>
    <tbody>
        <tr>
            <td><!-- Some custom item, like a checkbox or a group of headers --></td>
        </tr>
    </tbody>
</table>

angular.module("angular-table-restful-example").controller("interactiveExampleCtrl", ["$scope", "$filter", function($scope, $filter) {

  var vm = this;    

  vm.originalList = $scope.$parent.personList;

  vm.itemsPerPage = 5;
  vm.pagesToShow = 2;

  vm.add = add;
  vm.updateFilteredList = updateFilteredList;
  vm.loadTable = loadTable;

  function add() {
    vm.originalList.push({name: vm.nameToAdd});
    vm.updateFilteredList();
  };

  function updateFilteredList() {
    vm.tableData = $filter("filter")(vm.originalList, vm.query);
  };

  function loadTable(){
    vm.tableData = null;
    vm.tableData = $scope.$parent.personList;
  };

}])

Javascript

angular.module("angular-table-restful-example").controller("apiCtrl", ["$http", function($http) {
                                        var vm = this;    
  vm.changePage = changePage;
                                        //You must have a function specified on the 'at-change' attribute in order for the API Pagination to work.
                                        //Also the data that is passed to the 'deferred' must be an array with two more properties to work properly.
                                        //totalCount -> Total count of all items (e.g.: 3 pages with 5 items each, totalCount: 15).
                                        //pageNo -> The page that the API will retreive (same value that the pageInfo.pageNo has).
                                        function changePage(pageInfo, deferred){
                                        //The pageInfo parameters will have the following information:
                                        //  -> pageNo (int)
                                        //  -> pageSize (int)
                                        //  -> sort (string array, e.g.: '-name' will order the list by name DESC and 'name' will order the list by name ASC)
                                        var page = 'page' + pageInfo.pageNo + '.json';
    $http.get('api/' + page).then(function(successData){
                                        var data = prepareData(successData.data);
        deferred.resolve(data);
                                        console.log(data); //Also you can take a look at the api folder from this repo and see the json data.
    }, function(errorData){
                                        alert('Something went wrong with the API call :(');
    });
  }
                                        //This function could be an interceptor, just treat the API result
                                        function prepareData(data){
                                        var extractedData = [];
    extractedData = data.pageItems;
    extractedData.totalCount = data.totalCount;
    extractedData.pageNo = data.pageNo;
                                        return extractedData;
  }
}])
